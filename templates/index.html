How It Works

This tool uses bidirectional breadth-first search (BFS) to find the shortest path between two Wikipedia articles. Instead of searching in one direction, it searches from both the start and end pages simultaneously, meeting in the middle.

From the start page, the algorithm explores outbound links. From the end page, it follows backlinks. The search expands level by level until both sides find a common page.

Previously discovered connections are cached in a database as path segments. When you search for a path that shares segments with past searches, those segments are reused. Typical cache hit rates are 40-60% after warm-up.

The algorithm can find up to 5 different paths between two topics. Paths are filtered using Jaccard distance scoring—by default, each path must differ by at least 30% from previously found paths.

Frequently Asked Questions

How fast are the searches?

Most searches complete in 2-8 seconds. The bidirectional algorithm is significantly faster than single-direction search. Cached segments make repeated searches near-instantaneous (typical cache hit rate: 40-60%).

What is "six degrees of Wikipedia"?

Most Wikipedia articles can be connected in 3-5 links. This tool finds those shortest paths automatically.

Why do searches fail?

Searches timeout if the connection requires more than 6 hops. Obscure topics with few links, new articles, or different language editions may not connect. Try using more well-known intermediate topics.

What is the Knowledge Graph?

The Knowledge Graph visualizes cached path segments from previous searches. Each node is a Wikipedia page, sized by connections and colored by usage frequency.

Are searches saved?

Yes. All searches are saved in the Search History section. You can revisit, filter, or re-run them to see if new paths have been discovered.

How does multi-path discovery work?

The algorithm finds up to 5 different paths. It uses Jaccard distance scoring to ensure diversity—by default, each path must differ by at least 30% from previously found paths.
